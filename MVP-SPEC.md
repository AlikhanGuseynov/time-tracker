# Time Tracker MVP Spec

## Strategy for Working With Codex
- Iterate in small, focused prompts (e.g., "только каркас Electron+Angular", "только сервис трекинга времени").
- Каждый запрос должен включать контекст: выдержку из MVP-SPEC.md и перечень актуальных файлов.
- Ясно указывай, какие файлы существуют, какие можно менять и какие нужно создать.
- Используй формат: "Структура → файлы → требования → изменять только …".

## MVP1 Breakdown
1. **Каркас Electron + Angular**: окно Electron загружает Angular SPA, простые стили.
2. **Модель и сервис трекинга**: `TimeSession` + `TimeTrackerService` в Angular для старт/пауза/переключения категорий, всё в памяти.
3. **Главный экран**: кнопки категорий, текущий таймер, кнопка паузы, блок "Сегодня" с суммами по категориям.
4. **Хранение в JSON**: IPC + preload в Electron; файл `time-tracker-data.json` в `app.getPath('userData')`; Angular загружает/сохраняет сессии.
5. **Экран "Статистика"**: периоды (Сегодня/Вчера/Неделя/Месяц), группировка по категориям, суммы `HH:MM`, итог за период, простой роутинг/переключатель.
6. **Окно и tray**: закрытие скрывает окно, иконка в tray с меню Open/Exit; перед выходом закрывать активную сессию через IPC.

## Notes
- Angular стили простые (CSS без библиотек).
- Не использовать лишние зависимости — только стандартные для Electron/Angular.
- Писать код копируемым и минимальным.
